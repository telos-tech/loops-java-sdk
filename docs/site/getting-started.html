Getting Started

This guide will help you install and start using the Loops Java SDK in your project.

Prerequisites

Before you begin, ensure you have:

  • Java 21 or higher installed on your system
  • Maven 3.8+ or Gradle 7.0+ for dependency management
  • A Loops.so account with an API key
Getting an API Key: Log in to your Loops.so dashboard and navigate to Settings → API Keys to generate your API key.

Installation

Maven

Add the following dependency to your pom.xml:

<dependency>
    <groupId>com.telos</groupId>
    <artifactId>loops-java-sdk</artifactId>
    <version>1.0.0</version>
</dependency>

Gradle (Kotlin DSL)

Add the following to your build.gradle.kts:

implementation("com.telos:loops-java-sdk:1.0.0")

Gradle (Groovy)

Add the following to your build.gradle:

implementation 'com.telos:loops-java-sdk:1.0.0'

Basic Usage

Initialize the Client

Create a LoopsClient instance with your API key:

import com.telos.loops.LoopsClient;

public class Example {
    public static void main(String[] args) {
        // Initialize the client
        LoopsClient client = LoopsClient.builder()
                .apiKey("your-api-key-here")
                .build();

        // Use the client...
    }
}
Security Note: Never hardcode your API key in source code. Use environment variables or a secure configuration management system.

Using Environment Variables

It's recommended to store your API key in an environment variable:

LoopsClient client = LoopsClient.builder()
        .apiKey(System.getenv("LOOPS_API_KEY"))
        .build();

Set the environment variable before running your application:

# Linux/macOS
export LOOPS_API_KEY=your-api-key-here

# Windows (Command Prompt)
set LOOPS_API_KEY=your-api-key-here

# Windows (PowerShell)
$env:LOOPS_API_KEY = "your-api-key-here"

Creating a Contact

Create a new contact in Loops:

import com.telos.loops.contacts.ContactCreateRequest;
import com.telos.loops.contacts.ContactResponse;

ContactResponse response = client.contacts().create(
    ContactCreateRequest.builder()
        .email("user@example.com")
        .firstName("John")
        .lastName("Doe")
        .build()
);

if (response.success()) {
    System.out.println("Contact created with ID: " + response.id());
} else {
    System.out.println("Failed to create contact");
}

Sending an Event

Send an event to track user activity:

import com.telos.loops.events.EventRequest;
import com.telos.loops.events.EventResponse;

EventResponse eventResponse = client.events().send(
    EventRequest.builder()
        .email("user@example.com")
        .eventName("signed_up")
        .build()
);

if (eventResponse.success()) {
    System.out.println("Event sent successfully");
}

Sending a Transactional Email

Send a transactional email using a template:

import com.telos.loops.transactional.TransactionalRequest;
import com.telos.loops.transactional.TransactionalResponse;

TransactionalResponse emailResponse = client.transactional().send(
    TransactionalRequest.builder()
        .transactionalId("welcome_email")
        .email("user@example.com")
        .dataVariable("username", "John")
        .build()
);

if (emailResponse.success()) {
    System.out.println("Email sent successfully");
}

Working with Custom Properties

You can add custom properties when creating or updating contacts:

ContactResponse response = client.contacts().create(
    ContactCreateRequest.builder()
        .email("user@example.com")
        .firstName("Jane")
        .customProperty("plan", "premium")
        .customProperty("signupDate", "2026-02-10")
        .customProperty("lifetimeValue", 299.99)
        .build()
);

Error Handling

The SDK throws specific exceptions for different error scenarios:

import com.telos.loops.error.LoopsApiException;
import com.telos.loops.error.LoopsValidationException;
import com.telos.loops.error.RateLimitExceededException;

try {
    ContactResponse response = client.contacts().create(request);
} catch (LoopsValidationException e) {
    // Handle validation errors (invalid email, missing required fields, etc.)
    System.err.println("Validation error: " + e.getMessage());
} catch (RateLimitExceededException e) {
    // Handle rate limiting (SDK automatically retries, but you can add custom logic)
    System.err.println("Rate limit exceeded: " + e.getMessage());
} catch (LoopsApiException e) {
    // Handle other API errors
    System.err.println("API error: " + e.getMessage());
}
Automatic Retry: The SDK automatically retries requests that fail due to rate limiting with exponential backoff.

Asynchronous Operations

All operations have asynchronous variants that return CompletableFuture:

import java.util.concurrent.CompletableFuture;

// Asynchronous contact creation
CompletableFuture<ContactResponse> futureResponse =
    client.contacts().createAsync(request);

// Handle the result when it completes
futureResponse.thenAccept(response -> {
    System.out.println("Contact created: " + response.id());
}).exceptionally(throwable -> {
    System.err.println("Error: " + throwable.getMessage());
    return null;
});

Learn more about asynchronous operations in the Async Guide.
